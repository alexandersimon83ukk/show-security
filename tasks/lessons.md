# Lessons Learned

- Wenn der Nutzer eine bestimmte Testverteilung verlangt, sollten die Testfälle explizit so aufgebaut werden, dass die gewünschte Anzahl an Erfolgen und Fehlern klar nachvollziehbar ist.
- Coverage ist in Demo-Repositories am nützlichsten, wenn sie direkt im GitHub-UI sichtbar gemacht wird, nicht nur als Datei im Repository.
